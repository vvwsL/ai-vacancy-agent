# AI-агент подбора AI/ML вакансий по резюме

На вход — **твоё резюме**, на выходе — ранжированный топ-5 подходящих AI/ML вакансий с
объяснением: что совпало, что смущает, какие вопросы задать работодателю и какой приоритет
отклика. Артефакты каждого прогона: `report.md`, `trace.json`, `run.log`. Работает и без
API-ключа (детерминированный режим без LLM).

---

## Быстрый старт (что делать по шагам)

**1. Установить зависимости**
```bash
pip install -r requirements.txt
```

**2. (опционально) Подключить LLM** — нужен один бесплатный ключ:
```bash
cp .env.example .env
```
Открой `.env` и впиши ключ любого провайдера (проще всего **Gemini**:
https://aistudio.google.com/apikey). Без ключа всё равно работает — в режиме «без LLM».

**3. Положить резюме** в папку `data/` (формат pdf / docx / txt / md).
Для pdf/docx уже ставятся библиотеки из шага 1. Если пропустил — `pip install pypdf python-docx`.

**4. Запустить меню**
```bash
python -m src.menu
```
На Windows можно просто дважды кликнуть **`start.bat`**.

**5. В меню:**
```
  1) Резюме:            data/resume.txt      ← цифрой выбираешь файл из data/
  2) Источник вакансий: файл / telegram / pdf
  3) Провайдер LLM:     auto                 ← цифрой выбираешь (gemini/groq/...)
  4) Топ-N вакансий:    5                    ← вводишь число
  5) Режим разбора:     с LLM                ← y/n: с LLM или без (dry-run)
  6) > Запустить
  7) История прогонов
  0) Выход
```
Настрой пункты, нажми **6** — агент покажет, как понял твой профиль, попросит
подтвердить (Enter), и сделает разбор. Настройки сохраняются в `user_settings.json`.

**Источники вакансий (пункт 2) — можно выбрать несколько сразу** (отмечаешь `[x]`):
- **файл** — локальный `data/vacancies.json` (готовый пример, работает офлайн);
- **telegram** — публичные каналы `https://t.me/...` (вводишь ссылки по одной); читаем
  публичное превью канала, **LLM превращает посты в структурированные вакансии**;
- **pdf** — PDF/txt с вакансиями; парсим текст, **LLM режет на отдельные вакансии**.

Несколько источников объединяются в один список (дубли схлопываются). Через флаги:
`--sources file,telegram --tg-channels https://t.me/<канал>`.

**6. Посмотреть результат.** Путь печатается в конце прогона:
`runs/<дата>__<резюме>__<провайдер>/report.md`.
Либо через меню: **6) История прогонов** → выбрать прогон →
- **краткая сводка** (без LLM, прямо в терминале), или
- **открыть отчёт** в системном приложении (Windows/macOS/Linux сам откроет — напр. Блокнот).

> Ctrl+C во время действия — отмена и возврат в меню; на главном экране — выход.

## Запуск через флаги (без меню, для скриптов)

```bash
python -m src.main --resume data/resume.txt --provider gemini   # живой режим (LLM)
python -m src.main --dry-run --yes                              # без LLM, детерминированно
pytest                                                          # 23 теста
```
Флаги: `--resume --provider --top-n --input --dry-run --yes --output --config`.

## Что под капотом (для проверяющего)

### Поток
```
резюме (pdf/docx/txt/md)
   │  [resume.py]  парсер → текст                     (обычный код)
   ▼  [llm.py]     LLM извлекает профиль:             (LLM; rule-based fallback без ключа)
   │               роль, навыки, опыт, город, формат
   ▼  показ профиля + подтверждение
   ▼  [loader]     чтение/валидация/дедуп вакансий   (обычный код)
   ▼  [scoring]    hard-фильтр + взвешенный score      (обычный код)
   ▼  [llm.py]     агентный разбор топ-5 (tool-loop)  (LLM)
   ▼  [report.py]  report.md + trace.json + run.log
```
Уровень искомых вакансий определяется **детерминированно из опыта** (1–3 года → junior/middle).

### Где LLM/агент, а где обычная логика
| Обычная логика (детерминированно) | LLM/агент (принятие решения) |
|---|---|
| парсер файла резюме → текст | **извлечение профиля** из текста резюме |
| чтение/валидация/дедуп/нормализация вакансий | извлечение требований из описания |
| уровень из опыта, hard-фильтр, score, сортировка | сопоставление с критериями, риски |
| причины отсева по формальным признакам | вопросы к работодателю, приоритет (+override) |

Принцип: **арифметика и I/O — это код; суждение — это агент.** Score считается кодом
(воспроизводимо, объяснимо весами), LLM добавляет качественную оценку и может скорректировать
приоритет (`priority_override`) — расхождение со score видно в отчёте.

### Agent vs workflow — честно
Это **ограниченный агент**: ReAct-петля с нативным tool-calling (модель сама решает, какие из
3 инструментов вызвать: `get_full_description`, `compute_skill_overlap`,
`get_candidate_criteria`), но с лимитом итераций. Инструменты — детерминированный код; LLM их
только вызывает. Трасса вызовов — в `trace.json`.

### Надёжность
Деградация разбора (режим логируется в `trace.json`):
```
agent (tool-loop)  →  rule_based (без LLM, не падает никогда)
```
Обрабатываются: пустой/битый файл, не-объекты, пропущенные поля, дубли, длинное описание
(`as_is → structure_trim → hard_cut`), нет ключа (auto без LLM), мусорный ответ модели
(валидация Pydantic), сетевые ошибки/429, неподдерживаемый/пустой формат резюме, Ctrl+C.

### Limitations
- Источники: локальный файл, публичные Telegram-каналы (`t.me/s/` — последние ~20 постов,
  без авторизации), PDF/txt. Живой hh.ru не используется: API для соискателей закрыт
  15.12.2025; парсить HTML-страницы hh — против ToS, не делаем.
- Качество telegram/pdf-вакансий зависит от LLM-экстракции; без ключа — грубый fallback.
- Дедуп и матчинг навыков — строковые, без синонимов (Python ≠ py).
- LLM-режим воспроизводим по структуре, не дословно; полностью детерминирован режим без LLM.
- Веса scoring заданы экспертно (см. `config.yaml`), не калиброваны на данных.

### Что улучшил бы дальше
- семантический матчинг навыков/дедуп через embeddings;
- правка профиля «обычными словами» с перегенерацией через LLM;
- источники вакансий: открытые датасеты / экспорт с джоб-бордов через тот же загрузчик;
- калибровка весов scoring на размеченных данных.

## Провайдеры LLM
Все OpenAI-совместимы, бесплатны, один код: **Gemini**, **Groq**, **OpenRouter**,
**Cerebras**. `auto` берёт первый, для которого есть ключ в `.env`; если ключей нет — режим
без LLM. Ключи — только через `.env` (он в `.gitignore`).

## Структура
```
src/menu.py      интерактивное меню (лёгкий запуск) + история прогонов
src/main.py      CLI и оркестрация
src/resume.py    парсер резюме (pdf/docx/txt/md) → текст
src/fetch_telegram.py  чтение публичного превью Telegram-канала (t.me/s/)
src/llm.py       профиль + извлечение вакансий из текста + tool-loop агента + деградация
src/loader.py    чтение/валидация/дедуп/нормализация вакансий
src/scoring.py   hard-фильтр + взвешенный score + ранжирование
src/config.py    config.yaml + критерии (профиль из резюме или criteria.md)
src/report.py    report.md / trace.json / run.log
config.yaml      настройки: resume_path, веса scoring, top_n
data/            resume.txt (пример), vacancies.json (вакансии)
output/          закоммиченный эталонный пример прогона
runs/            история прогонов (создаётся при запуске; в .gitignore)
tests/           23 теста: loader, scoring, llm-fallback, resume, telegram
```
